Smart Planter System Backend API Documentation

This document defines the RESTful and streaming API specifications for the Smart Planter system backend controller deployed on Raspberry Pi 4. It provides endpoints for sensor data retrieval, pump hardware execution, automated AI workflow scheduling, system configuration management, and live MJPEG video streaming.

1. Global Conventions

  • Base URL: http://<pi-ip-address>:<port>
  • Data Format: All request and response bodies must use the application/json media type.
  • Time Format: All timestamps adhere strictly to the ISO 8601 extended format in UTC (e.g., 2026-06-14T06:00:00Z).

2. Sensor Data Endpoints

2.1 Get Sensor Readings

Retrieves sensor telemetry data. The system automatically collects and stores sensor readings at a configurable interval (default: every 3 seconds). This endpoint supports two modes of operation:

  • Real-time mode (default): Without query parameters, returns the latest live reading directly from hardware sensors.
  • Historical mode: When the start_time parameter is provided, returns all stored historical readings from that timestamp onward.
GET /api/v1/sensors/current
Query Parameters
Parameter Required Type Description
start_time No String (ISO 8601) If provided, returns all historical sensor records from this UTC timestamp onward. If omitted, returns the latest real-time reading from hardware.
Response Fields (Single Reading)
Field Name Type Description
timestamp String The exact UTC time when the sensor payload was gathered.
temperature_celsius Float Ambient temperature measured in degrees Celsius (°C).
atmosphere_hpa Float Atmospheric pressure value measured in hectopascals (hPa).
soil_moisture_percent Float Volumetric water content in the soil expressed as a percentage (0.0 - 100.0).
Success Response — Real-time Mode (200 OK)
{
  "timestamp": "2026-06-14T06:15:00Z",
  "temperature_celsius": 25.3,
  "atmosphere_hpa": 1011.5,
  "soil_moisture_percent": 45.2
}
Success Response — Historical Mode (200 OK)

Returned when the start_time query parameter is provided.

{
  "total_records": 120,
  "records": [
    {
      "timestamp": "2026-06-14T06:00:00Z",
      "temperature_celsius": 24.8,
      "atmosphere_hpa": 1010.3,
      "soil_moisture_percent": 48.1
    },
    {
      "timestamp": "2026-06-14T06:00:03Z",
      "temperature_celsius": 24.9,
      "atmosphere_hpa": 1010.4,
      "soil_moisture_percent": 47.8
    }
  ]
}

3. Actuator Control Endpoints

3.1 Execute Single Manual Watering Task

Triggers the relay board to activate the water pump actuator for a specified duration. The pump automatically switches off when the duration expires.

POST /api/v1/pump/action
Request Body Parameters
Parameter Required Type Description
action Yes String Must be explicitly set to "run".
duration_milliseconds Yes Integer The operational runtime for the water pump in milliseconds.
Success Response (202 Accepted)
{
  "status": "executing",
  "message": "Pump turned on, will automatically turn off after 5000ms.",
  "estimated_end_time": "2026-06-14T06:15:05Z"
}
Error Response (400 Bad Request)

Returned if the requested duration exceeds the hardware security thresholds defined within the system configuration.

{
  "error": "EXCEEDS_MAX_DURATION",
  "message": "Action denied. Duration exceeds the safety threshold specified in configuration."
}

3.2 Get Pump Status

Fetches the current real-time electrical state and operational health metrics of the water pump hardware module.

GET /api/v1/pump/status
Success Response (200 OK)
{
  "is_running": false,
  "last_executed_time": "2026-06-14T03:00:00Z",
  "last_duration_milliseconds": 4000,
  "hardware_healthy": true
}

3.3 Emergency Stop Pump

Immediately interrupts the driving power to the water pump hardware, instantly canceling any automated schedules or manual tasks currently executing.

POST /api/v1/pump/stop
Success Response (200 OK)
{
  "status": "stopped",
  "message": "Pump stopped immediately."
}

4. AI Analysis & Scheduling Endpoints

4.1 Manually Trigger AI Analysis

Instructs the backend to immediately gather current physical sensor data and capture a single frame from the live video stream. This unified dataset is compiled and processed by the AI core to recalculate the plant growth status and optimize upcoming irrigation schedules.

POST /api/v1/ai/trigger
Success Response (200 OK)
{
  "triggered_at": "2026-06-14T06:16:00Z",
  "status": "completed",
  "ai_decision_summary": "Analysis based on image and physical sensors. Foliage structure appears nominal. Soil moisture at 45.2% is stable. No immediate watering required.",
  "schedules_updated": true
}

4.2 Get Upcoming Watering Schedule

Returns the queue of planned future watering operations generated through automated cron evaluations or manual AI interventions.

GET /api/v1/schedule/upcoming
Success Response (200 OK)
{
  "generated_at": "2026-06-14T06:16:00Z",
  "ai_decision_summary": "Soil moisture is declining slowly. Scheduling a maintenance watering session for this evening.",
  "schedules": [
    {
      "schedule_id": "sch_20260614_1800",
      "planned_time": "2026-06-14T18:00:00Z",
      "duration_milliseconds": 3500,
      "executed": false
    }
  ]
}

4.3 Get Execution Records and Decision History

Retrieves a paginated collection of historical watering event executions and associated soil telemetry changes.

GET /api/v1/schedule/history
Query Parameters
Parameter Type Default Description
limit Integer 10 Maximum number of log items to return in the payload.
offset Integer 0 The entry index offset position for pagination.
Success Response (200 OK)
{
  "total_records": 12,
  "records": [
    {
      "executed_at": "2026-06-14T00:00:02Z",
      "trigger_type": "AI_SCHEDULED",
      "duration_milliseconds": 3500,
      "status": "SUCCESS",
      "soil_moisture_before": 38.5,
      "soil_moisture_after": 52.1
    }
  ]
}

5. System Configuration Endpoints

5.1 Get Current System Configuration

Fetches global runtime parameters regulating hardware sampling loops and automated execution parameters.

GET /api/v1/system/config
Success Response (200 OK)
{
  "sensor_read_interval_seconds": 3,
  "ai_evaluation_cron": "0 8,18 * * *",
  "safety_max_duration_milliseconds": 15000
}

5.2 Update System Configuration

Modifies the system operational variables. Supports full or partial updates of the configuration object.

PUT /api/v1/system/config
Request Body Parameters
Parameter Required Type Description
sensor_read_interval_seconds No Integer Interval in seconds for background sensor data collection and storage (default: 3). Changes take effect immediately.
ai_evaluation_cron No String Standard crontab string regulating automated AI execution.
safety_max_duration_milliseconds No Integer Hard threshold preventing excessive manual pump activity.
Success Response (200 OK)
{
  "status": "success",
  "message": "Configuration updated successfully."
}

6. Camera Streaming Interface

6.1 Get Live Video Stream

Exposes the continuous video stream generated directly by the hardware camera drivers. Outputs a persistent stream using standard MJPEG format.

GET /api/v1/camera/stream
Response Headers
Content-Type: multipart/x-mixed-replace; boundary=frame
Cache-Control: no-cache, private
Success Response (200 OK)

Returns a continuous binary stream of individual JPEG frames separated by the boundary string specified in the header. Web panels can map this endpoint directly into the src attribute of standard <img> elements.
